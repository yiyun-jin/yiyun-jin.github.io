WindSense: An Interactive Device for Real-Time Wind Simulation


1. Background: From Early Sailing to Wind Simulation

I have been learning sailing since elementary school, and for me, wind is not only the essential driving force in competition but also a window to understanding nature. However, it is difficult to experience the changing qualities of wind during indoor training. To address this, I designed an interactive device: an outdoor wind sensor collects real-time data and transmits it indoors. A controller then drives a motor to adjust a fan, recreating the same wind conditions inside. This not only deepened my understanding of wind but also provided a new way to support sailing practice.

2. Concept: Extending Wind into Interactive Experiences

During the process, I realized the device had broader potential. Wind is not only a natural phenomenon we observe but also a medium that carries emotions and memories. For example, during a video call, if one person at the seaside uses the device to capture and transmit wind data, the other person can feel the same breeze through their fan, sharing the atmosphere and environment across distance. This adds a new sensory dimension to communication and makes emotions more tangible. In the future, such a design could even be applied in VR gaming or immersive interactive experiences.

3. Interactive Wind Simulation System

3.1 System Design

This system is a wireless data acquisition and device control system, covering data acquisition, wireless transmission, device control, and power supply modules. The overall workflow is as follows:

Data Acquisition Module (Anemometer + Power)

Function：Measure environmental wind speed.
Implementation:

  • The anemometer converts wind changes into electrical signals using its internal sensing element.
  • The signal is processed by a built-in chip and output via Modbus protocol.
  • A power bank provides continuous power to the anemometer.

Data Processing & Transmission Module (ESP32 + Power)

Function: Receive wind data, process it, and transmit wirelessly.
Implementation:

  • ESP32 connects to the anemometer via UART and a serial-to-RS485 module.
  • The Modbus protocol is used to parse the data packet and extract wind speed.
  • After data validation, ESP32 sends the data via Wi-Fi (TCP/IP protocol).
  • A power bank provides stable power to ESP32 for reliable operation.

Wireless Receiving & Control Module (ESP32 + Power)

Function: Receive wireless data, generate control signals, and drive the fan matrix.
Implementation:

  • The second ESP32 receives wireless wind data via Wi-Fi.
  • It runs a pre-programmed control algorithm to generate PWM signals.
  • A step-down module converts 24V to 5V to power the ESP32.

Actuation Module (Fan Matrix + Power)

Function: Adjust fan operation (speed, on/off) based on control signals.
Implementation:

  • The fan matrix receives PWM signals and adjusts speed accordingly.
  • DC24V 60A power supply provides energy for the fans.
  • AC220V mains electricity is converted to DC24V to meet the fan matrix’s power requirements.

Overall Workflow Summary
Anemometer measures wind speed → ESP32 1 processes and transmits data wirelessly → ESP32 2 receives data and generates control signals → Fan Matrix operates according to the control signals.

4.3 Conclusion

  • Achievement of Core Design Goal
    The core goal of our system is to “collect real wind speed data outdoors, process it, and control the indoor fan matrix to simulate outdoor wind conditions indoors.” Looking at the workflow of all modules, this goal has basically formed a complete loop.

  • Data Acquisition: The outdoor anemometer can accurately measure wind speed. It outputs standardized wind data through the RS-485 interface using the Modbus-RTU protocol, avoiding interference from analog signals. A power bank supplies the anemometer, so it works reliably even without a fixed power source outdoors, providing accurate reference data for indoor simulation.

  • Data Transmission: The wind data is reliably transmitted from outdoors to indoors. A serial-to-RS485 module solves the interface problem between the ESP32 and the anemometer, and the ESP32 sends the processed wind data wirelessly via Wi-Fi. This avoids running complicated wires between outdoors and indoors. The protocol is chosen to balance stability and real-time performance, allowing the indoor controller to quickly respond to changes in wind speed.

  • Control & Execution: Indoor fans adjust according to the outdoor wind speed. The indoor ESP32 receives the data and generates control commands based on a preset algorithm. A DC 24V 60A power supply provides enough power for the fan matrix, which adjusts fan speed and on/off states according to outdoor wind. For example, fans run slowly when the outdoor wind is light and run fast when it is strong, reproducing a wind condition indoors that matches the outdoor environment.

5. Limitations & Further Work

Limitations:

  • Resonance Issue: At certain wind speed ranges, fan columns vibrate due to resonance; below or above this range, the system is stable.

  • Wind Flow Pattern: Fan-generated wind is spiral-shaped, not entirely natural; a possible improvement is using straws to build a flow-straightening wall.

Future Improvements:

  • Optimize mechanical design to reduce resonance.
  • Use flow-straightening devices to improve wind naturalness.
